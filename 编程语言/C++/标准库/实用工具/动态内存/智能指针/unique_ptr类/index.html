unique_ptr类

一、基本操作​

unique_ptr和shared_ptr不同，某个时刻只能有一个unique_ptr指向一个给定对象，当unique_ptr被销毁时，它所指向的对象也将被销毁。

从模型上来说，虽然并不清楚其具体的实现，但可以适当假设unique_ptr并没有引用计数和弱引用计数，甚至可以没有控制块，而只包含一个指针直接指向分配的动态内存，而其析构函数执行的操作为首先释放动态内存，随后销毁unique_ptr本身。

其支持的操作如下表所示：

操作效果
unique_ptr<T> up空unique_ptr，可以指向类型为T的对象
unique_ptr<T, D> up(d)空unique_ptr，指向类型为T的对象，用类型为D的对象d代替delete
unique_ptr<T> up(p)用内置指针p初始化up，explicit的，不能拷贝
unique_ptr<T, D> up(p, d)用内置指针p初始化up，并用类型为D的对象d代替delete
*p解引用p，获得它指向的对象
p->mem等价于(*p).mem
p.get()返回p中保存的指针
swap(p, q)
p.swap(q)		交换p和q中的指针
u = nullptr释放u指向的对象，将u置为空
u.release()u放弃对指针的控制权，返回指针，并将u置为空
u.reset()
u.reset(q)
u.reset(nullptr)		释放u指向的对象。
如果提供了内置指针q，令u指向这个对象；
否则将u置为空

除此之外，unique_ptr还定义了一个类型转换：

  • 从unique_ptr到bool值的转换：当unique_ptr对象指向一个对象时，则为true，否则为false

二、操作解析​

1. 初始化​

与shared_ptr不同，没有类似make_shared_ptr的标准库函数返回一个unique_ptr，当我们定义一个unique_ptr时，需要将其绑定到一个new返回的指针上。类似shared_ptr，初始化unique_ptr必须采用直接初始化形式：

unique_ptr<double> p1;
unique_ptr<int> p2(new int(42));

由于一个unique_ptr拥有它指向的对象，因此unique_ptr不支持普通的拷贝或赋值操作，例如：

unique_ptr<int> p3(p2);         // 错误：不能拷贝
p3 = p2;                        // 错误：不允许赋值

当然，即便如此仍然可以使两个unique_ptr指向同一块内存：

int *pi = new int(42);
unique_ptr<int> p1(pi);
unique_ptr<int> p2(pi);

但和shared_ptr一样，如此混用内置指针和智能指针会导致很多问题，最好还是不要这样做。

2. 特殊的拷贝​

当传递unique_ptr作为参数类型或返回unique_ptr作为返回值类型时，此时的拷贝是有效的：

unique_ptr<int> clone(int p) {
    return unique_ptr<int> (new int(p));
}

unique_ptr<int> clone(int p) {
    unique_ptr<int> ret(new int(p));
    return ret;
}

在此情况下，编译器执行一种特殊的“拷贝”，即移动拷贝。

3. 传递删除器​

和shared_ptr不同，unique_ptr传递删除器之前需要在实例化步骤显式给出删除器的类型，而shared_ptr不需要。

我们必须在尖括号中unique_ptr指向类型之后提供删除器类型，在创建或reset一个unique_ptr类型的对象时，必须提供一个指定类型的可调用对象（删除器）：

void f(destination &d)
{
    connection c = connect(&d);
    unique_ptr<connection, decltype(end_connection)*>
        p(&c, end_connection);
}

需要显式提供删除器的场景仍然是在托管一些没有析构函数的类型的对象或管理不能由delete释放的内存时有用。
