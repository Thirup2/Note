变量属性

一、名字的作用域​

在一个程序中可能会有多个相同的名字，但却是不同的实体（虽然并不建议这样做）。那么我们每次使用这个名字它所代表的实体是哪一个就是由这些名字的作用域决定的。

名字的作用域从名字的声明语句开始，以声明语句所在的作用域末端结束。不允许在同一作用域声明或定义同名的变量。

1. 全局作用域​

定义于所有花括号之外的变量，它们拥有全局作用域，在整个程序的范围内都可使用。

2. 块作用域​

定义于花括号内的变量，无论是函数块，还是语句块，其中定义的变量都属于块作用域。其作用域只限于该块及其下层嵌套作用域

3. 语句作用域​

可以在if、switch、while和for语句的控制结构内定义变量。这些变量只在语句的范围内可见，一旦语句结束，变量也就超出其作用范围了。

4. 类作用域​

类的定义体是一个独立的作用域称为类作用域。

嵌套作用域​

作用域可以彼此包含，被包含的作用域称为内层作用域，包含着别的作用域的作用域称为外层作用域。

外层作用域中定义的名字在内层作用域中也可以访问，同时也允许在内层作用域中重新定义外层作用域已有的名字，从内存作用域中同名变量声明语句开始到内层作用域结束，外层作用域中的相同名字将被屏蔽。

作用域操作符​

::为作用域操作符，当左侧为空时，表示访问全局作用域中的变量名，如：

int a = 20;
int main()
{
    int a = 10;
    std::cout << a << std::endl;    // 此处 a 为 10
    std::cout << ::a << std::endl;  // 此处 a 为 20
    
    return 0;
}

二、变量的链接属性​

变量的作用域通常决定一个名字在同一个文件中所表示的不同实体，而变量的链接属性则决定同一个名字在不同文件中所表示的实体。

链接属性一共有 3 种，外部链接（external）、内部链接（internal）和无链接（none）。

其中无链接属性的标识符总是被当做单独的个体，也就是说该标识符的多个声明被当作不同的独立实体。

属于 internal 链接属性标识符在同一个源文件内的所有声明中都指向同一个实体，但位于不同源文件的多个声明则分属不同的实体（需要注意的是同一个源文件中只应该有一个定义，其余都是仅声明）。

而属于 external 链接属性的标识符不论声明多少次，位于几个源文件都表示同一个实体（注意也只有一个定义，其余都是仅声明）。
