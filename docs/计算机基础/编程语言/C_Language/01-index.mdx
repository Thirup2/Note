## 文档简介 {#intro_of_doc}

此文档是关于 C 语言从 C90 标准到 C18 标准的全部内容，其组织方式与 C 语言标准文档采用相同的结构，主要分为四个部分：术语、环境、语法和标准库。其中最重要的语法部分包括最基本的词法元素、类型转换、表达式等内容。

本文档是一个高度总结性的 C 语言的知识文档，即本文档将按照体系化的组织方式进行编写，而不是按照从已知到未知的这样一个循序渐进的组织方式进行编写。故本文档并非教程类的 C 语言文档，而算是一个手册类的文档，从而本文档将不适合寻求 C 语言教程的人。（如果你在寻找 C 语言教程，那我推荐你可以去看下方本文档的[参考资料](#reference)中的相关书籍和文档）。

本文档为作者纯手打，只有我 1 个人的参与难免会有疏漏和理解不到位的地方，如果对于本文档你有任何疑问和建议，可以通过页脚的联系方式联系我，非常感谢来自任何人的反馈。

最后，希望你都可以通过本文档而有所收获。



## C 语言标准 {#C_STD}

在本文档开始编写时，C 语言的 ISO 标准为 C18，以往的标准有 C90、C99、C11。

本文档在编写过程中将以 C90 为准，当然其实该标准还有一些其他的名字如 C89、ANSI C 等都是指的该标准，在本文档中将一直使用 C90 称呼。在 C90 之后的其他 ISO 标准中较 C90 更新的内容将在文档对应地方额外给出（具体样式如下）。也就是说，如果在正文中（没有特别指出的时候）看到诸如 “C 语言就是……”、“C 语言可以……” 等语句时都应该理解为 “C90 就是……”、“C90 可以……” 等等。
