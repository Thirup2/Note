有时候我们可以会创建一个`const`修饰的类类型对象，例如：

```cpp
const string("hello world");
```

从宏观上看，我们希望它和一个`const`修饰的内置类型具有一样的特性。

那么从微观上看呢？一个用`const`修饰的类类型对象其内部是否发生的变化呢？

实际上并没有，当我们用`const`修饰一个类类型对象时，只需注意两点：

- 如果直接操纵类对象中`public`的数据成员，`const`修饰的类类型对象则会禁止这样的操作。但包括`private`的数据成员在内，所有的数据成员类型并不会随着`const`的修饰而改变
- 如果不直接操纵类对象中的数据成员，一个`const`的类类型对象只能调用`const`成员函数，而这样的函数通常不会修改数据成员的值。
