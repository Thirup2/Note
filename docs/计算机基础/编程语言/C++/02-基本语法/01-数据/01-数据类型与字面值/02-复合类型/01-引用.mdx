## 一、声明与初始化

**引用**为对象起了另外一种名字，通过将声明符写成`&d`的形式来定义引用类型，其中`d`是引用的名字：

```cpp
int ival = 1024;
int &refVal = ival;		// refVal 指向 ival（是 ival 的另一个名字）
// error-next-line
int &refVal2;			// 错误：引用必须被初始化
```

**要点一**：本节标题只有声明和初始化，而并没有变量的定义过程，这是因为引用本身并非一个对象，它不需要分配内存，它只是被引用对象的另一个名字

**要点二**：引用一旦与某个对象绑定，则无法再绑定到另一个对象，所以在声明一个引用的时候就需要对其进行初始化，将其与一个对象进行绑定。

**要点三**：由于引用本身没有实体，所以它的初始化过程和变量的初始化过程也不相同，一个引用的初始化是将引用名与一个对象绑定起来而并非往某个内存中写入值



## 二、引用的使用

- 由于引用只是一个别名，所以对一个引用进行的所有操作都是在与之绑定的对象上进行的，例如：

  ```cpp
  refVal = 2;			// 把 2 赋给 refVal 指向的对象，此处即是赋给了 ival
  int ii = refVal;	// 与 ii = ival 执行结果一样
  ```

- 由于引用本身不是一个对象，所以不能定义引用的引用：

  ```cpp
  int &refVal3 = refVal;		// refVal3 是绑定到了 refVal 绑定的对象上，而不是绑定到了 refVal 上
  int i = refVal3;			// i 被初始化为 ival 的值
  ```

- 引用的类型要和与之绑定的对象的类型严格匹配（特殊情况除外）。而且，引用只能绑定在对象上，而不能与字面值或某个表达式的计算结果绑定在一起：

  ```cpp
  // error-next-line
  int &refVal4 = 10;		// 错误：引用类型的初始值必须是一个对象
  double dval = 3.14;
  // error-next-line
  int &refVal5 = dval;	// 错误：引用类型与被引用对象类型不匹配
  ```



## 三、复杂引用类型

### 1. 对指针的引用

```cpp
int i = 42;
int *p;

int *&r = p;		// r 是一个对指针 p 的引用
r = &i;				// 将 p 指向 i
*r = 0;				// 向 i 赋值 0
```

对于指针的引用的声明语句，我们可以如下理解：

- `r`首先与`&`结合，表示这是一个引用
- 再和`*`结合，表示这是一个对指针的引用
- 最后和`int`结合，表示这是一个对指向`int`类型的指针的引用
