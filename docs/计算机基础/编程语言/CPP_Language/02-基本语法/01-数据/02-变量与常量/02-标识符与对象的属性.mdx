## 简介

一个变量有两个必要因素，一是用于标识它的标识符，另一个它所占用的内存（对象）。

对于标识符来说，我们需要关注其**作用域**和**链接**，对于对象来说，我们需要关注其**存储类型**和**生存期**。



## 一、标识符的作用域

不论是在程序的什么位置，使用到的每个名字都会指向一个特定的实体：变量、函数、类型等。然而，同一个名字如果出现在程序的不同位置，也可能指向的是不同的实体。

**作用域**是程序的一部分，在其中名字有其特定的含义。C++ 语言中大多数作用域都以花括号分隔。

标识符的有效区域始于名字的声明语句，以声明语句所在的作用域末端为结束。

:::info 作用域与名字的有效区域

需要注意的是作用域是程序的一部分，通常从左花括号到右花括号。一个名字的有效区域并不一定是整个作用域，如上所述，它通常是从名字的声明语句开始。

:::

### 1. 全局作用域

定义于所有函数体之外的名字具有**全局作用域**，全局作用域内的名字在整个程序的范围内都可使用。



### 2. 块作用域

在函数的左右花括号之间以及复合语句的左右花括号之间是**块作用域**，从声明具有块作用域的变量开始直到块结束为止都可以访问它。




### \*-1 嵌套作用域

作用域能够彼此包含，被包含的作用域称为**内层作用域**，包含着别的作用域的作用域称为**外层作用域**。

作用域中一旦声明了某个名字，它所嵌套着的所有作用域中都能访问改名字。

同时，允许在内层作用域中重新定义外层作用域已有的名字，如果像这样做，那么在内层作用域中该定义语句之后，对于该标识符的访问将不再是外层作用域中该标识符对应的对象，外层的名字将被内层的名字屏蔽，直到退出内层的作用域。



### \*-2 作用域操作符

使用**作用域操作符**（`::`）可以指定名字所在的作用域。其语法一般是

```c++
namespace::name
```

其中`namespace`是命名空间或作用域的名字，`name`则是要访问的对象的名字。

特殊地，全局作用域本身没有名字，所以`::name`实际上是在访问全局作用域中的`name`对象。当全局作用域中的对象被内层作用域中的同名对象屏蔽之后，可以使用该方法访问全局作用域中的对象。
