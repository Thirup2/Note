---
title: 继承与动态绑定
---

在 C++ 中，**面向对象程序设计**的核心思想是数据抽象、继承和动态绑定。通过数据抽象，我们可以将类的接口与实现分离；使用继承，可以定义相似的类型并对其相似关系建模；使用动态绑定，可以在一定程度上忽略相似类型的区别，而以统一的方式使用它们的对象。

## 一、继承

一个类可以继承自另一个类。被继承的类称为**基类**，继承基类的类称为**派生类**。基类负责定义在整个层次关系中所有类共同拥有的成员，而每个派生类定义各自特有的成员。

对于处于继承关系中的基类，其成员除了普通成员外，还包含一些**虚函数**，表示希望其派生类各自定义适合自身的版本。

派生类必须通过**类派生列表**明确指出它是从哪个（哪些）基类继承而来的。在类派生列表中出现的基类还需要使用一个访问说明符。派生类如果对基类中的虚函数进行了重新定义，那么必须进行声明，派生类可以在这样的函数之前加上`virtual`关键字，或者在该函数的形参列表之后增加一个`override`关键字。



## 二、动态绑定

在 C++ 中，当我们使用基类的引用（或指针）调用一个虚函数时将会发生动态绑定。

**动态绑定**是在运行时决定对于虚函数的调用究竟使用的是哪一个类的实现，当然，这些类都应该是调用该函数的类型的派生类。
