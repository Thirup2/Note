C++

文档简介​

此文档较为全面地总结了 C++ 的相关内容，总体内容基本包括 4 个方面。首先是基本语法部分，包括数据、表达式、语句、函数这 4 个方面，这 4 个方面几乎是每门编程语言都具备的语法要素；其次是 C++ 类的相关内容，算是 C++ 面向对象特性里的一个非常重要的组成部分；第三是 C++ 标准库，大部分语言都会有自己的标准库，而 C++ 的标准库则是出了名的复杂；最后一部分就基本上是模板与泛型编程相关的内容了，模板的使用可以大大提高类的可重用性，同时也是一个比较难的部分。

另外，本文档主要按照知识体系的组织方式进行编写，而不是按照从已知到未知的组织方式进行编写，故本文档从本质上来说并不是一个教程类文档，而是一个 C++ 的编程参考。如果你是在寻找一个 C++ 教程，那我并不建议你直接看本文档，但我推荐你阅读本文档的参考书籍。

本文档为作者纯手打，只有我 1 个人的参与难免会有疏漏和理解不到位的地方，如果对本文档有任何的疑问或者建议，可以通过页脚的联系方式来联系我，非常感谢来自任何人的任何反馈。

最后，提一嘴关于 C++ 的学习方法。相信很多人和我一样都是先学了 C 语言然后再学的 C++。C++ 在语法层面上和 C 语言极其相似这是毫无疑问的，但我的建议是在学习 C++ 的时候最好暂时忘记 C 语言，因为你会发现它们实际上在某些方面还是存在区别的，比如 C++ 的类可以用struct关键字，而 C 语言的结构也使用struct关键字，但要是你一直用 C 语言的struct语法去理解 C++ 的类，你会发现理解起来反而更难了。当然，在很多地方还是可以用 C 语言作为参考去理解 C++ 的，将多个语言进行对比学习也非常有利于理解 ”编程语言“ 本身，但具体在什么时候参考 C 语言，而什么时候不应该参考 C 语言我想这就需要你在学习的过程中自己去判断了。

C++ 标准​

在本文档开始编写时，C++ 的 ISO 标准还仅仅只是 C++20，而实际上有一部分编译器支持的标准已经到 C++23 了。

虽然如此，但本文档在编写过程中仍将以 ISO 标准为准，而由于 C++11 较之前的标准变动较大，故以 C++11 的 ISO 标准为基本标准，后续的所有标准中更新了的内容将额外给出（具体样式如下）。也就是说，你可以在本文档中看到大部分不是特别列出的的 “C++ 就是……”、“C++ 可以……” 等语句都应该转换为 “C++11 就是……”、“C++11 可以……” 等等。这表示如果你使用 C++11 之前的 C++ 版本或许将与本文档内容产生冲突。
